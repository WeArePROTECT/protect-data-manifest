# Finding PROTECT data with the manifest

The PROTECT data manifest is a file-based catalog at
`/usr2/people/protect/protect-data-manifest`. It describes data **in place** — it tells you where
data lives, what's in it, and how to connect it; it does not hold the data itself.

## The protocol — follow these steps

1. **Read the catalog: `manifest/INDEX.md`.** It lists every collection with a one-line `summary`,
   `tags`, `keywords`, path, and access status, plus a **facet index** (`tag → collections`) at the
   bottom. To find candidates fast, match the user's request against keywords/tags, or `grep` the
   facet index (e.g. AMR, CLINICAL, PHENOTYPE, GENOME, MIND, LINKAGE, LAKEHOUSE_EXPORT).

2. **Open the matched collection(s)** under `manifest/collections/<id>/`:
   - **`CARD.md`** first — it has the plain-language description, **join recipes**, caveats, and an
     **"Example questions this answers"** section that often maps the user's exact question to the
     files/columns/joins to use.
   - **`dataset.yaml`** for ground truth — exact file paths (relative to `collection_root`), every
     column with its inferred type, row counts, sizes, and machine-verified freshness.
   - **For "which file/run is newest?"** read the descriptor's **`latest_resource`** (newest data
     file) and **`latest_subdir`** (newest dated run/export), plus each resource/subdirectory's
     `mtime`. Don't infer recency from filenames — cards deliberately defer "what's current" to these
     machine-refreshed fields.

3. **To connect datasets, use the join keys.** The canonical join layer is
   `patient_sample_isolate_linkage`: `ASMA_id` (isolate) ↔ `patient_id` (patient) ↔ `sample_id`
   (sample). A card's `related` field lists the collections it commonly joins with. Plan
   cross-dataset queries by routing through the linkage table.

4. **Honor access.** Each collection's `access.status` is `open` or `gated` (derived from real
   filesystem permissions). If it's `gated`, tell the user the data exists and what's in it, but
   that they'll need access — name the group / who to ask from `access.request_to`. Never imply the
   user can already read gated bytes.

5. **Trust the descriptor over any prose.** `dataset.yaml` is regenerated from the live filesystem;
   a human `CARD.md` (or an owner's federated manifest) can lag. If `owner_manifest` is present, it
   can be a richer source, but check its claimed date against the descriptor's `source_mtime_latest`.

## Worked example (Jake's request)
> "Take the ASMA collection and filter it for AMR gene predictions and growth phenotypes."

- `grep` the facet index for `AMR` → `asma_genomics`; for `PHENOTYPE` → `asma_phenotyping`.
- Open `asma_genomics/CARD.md` + `dataset.yaml` → `amrfinder.tsv`, keyed by `ASMA_id`.
- Open `patient_sample_isolate_linkage` → join `ASMA_id` → `patient_id` / `sample_id`.
- Open `asma_phenotyping` → growth phenotypes, also keyed by `ASMA_id`.
- Result: `asma_genomics` (AMR) ⨝ linkage (`ASMA_id`→patient) ⨝ `asma_phenotyping` (growth).

## Notes
- The catalog is regenerated by `crawler/crawl.py` + `crawler/build_index.py`; if something looks
  missing or stale, those are what refresh it.
- The standard each collection conforms to is in `templates/README.md`.
